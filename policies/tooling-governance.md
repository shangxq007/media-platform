## Purpose

Govern tool selection, ownership, and lifecycle for media-platform.

## Tool Lifecycle

| Stage | Description | Approval |
|-------|-------------|----------|
| PLANNED | Documented, not yet implemented | TOOLING.0 |
| ACTIVE | In use, maintained | TOOLING.0 |
| AVAILABLE | Installed, not actively used | TOOLING.0 |
| EXPERIMENTAL | Under evaluation | TOOLING.0 |
| DEFERRED | Explicitly postponed | ADR |
| DEPRECATED | No longer recommended | ADR |

## Tool Introduction Rules

1. New tool must be documented in tooling-taxonomy.md
2. New tool must have assigned owner
3. New tool must have defined agent permissions
4. New tool must not duplicate existing tool in same layer
5. New tool must not violate architecture constraints
6. Production tools require human approval

## Tool Deprecation Rules

1. Deprecation must be documented
2. Replacement must be identified
3. Migration path must be defined
4. Human approval required for production tools
