## Rules for Coding Agents

1. **Do not update dependencies** unless task explicitly allows
2. **Dependency PRs must not mix** business code changes
3. **Dependency PRs must include:**
   - Reason for update
   - Changelog links if available
   - Test result
   - Risk classification (patch/minor/major/security)
   - Rollback note
4. **Major updates** require human review
5. **Generated dependency PRs** stay draft or needs-review until approved

## Review Packet Section

Add to Review Packet:
```
## Dependency Changes
- Dependency changes: YES / NO
- Direct dependencies changed: [list]
- Transitive risk noted: [details]
- Security update: YES / NO
- Major update: YES / NO
- Tests run: [result]
- Rollback plan: [plan]
- Human approval required: YES / NO
```

## Local Guardrails Integration

- Local guards do not replace dependency governance
- Dependency file changes should be clearly reported
- Optional warning when dependency manifests change outside dependency tasks
