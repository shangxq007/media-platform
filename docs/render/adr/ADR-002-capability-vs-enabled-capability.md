# ADR-002: 能力矩阵 vs 开放能力

## 状态

已接受

## 日期

2026-06-11

## 背景

当前系统没有区分"理论支持的能力"和"当前允许调度的能力"，导致：
- 能力矩阵误导调度器
- 测试计划不准确
- 文档和实际行为不一致

## 决策

引入三种能力列表：

1. **declaredCapabilities**：该工具理论支持的能力，用于能力矩阵、文档和规划
2. **enabledCapabilities**：当前阶段允许系统调度的能力
3. **disabledCapabilities**：理论支持但当前不启用的能力
4. **notFor**：明确不应该由该 provider 承担的能力

**调度器只能基于 enabledCapabilities 进行匹配，不能基于 declaredCapabilities 调度。**

## 后果

- 能力矩阵不等于开放能力
- 调度器行为更可预测
- 文档和实际行为一致
- 测试更有针对性

## 替代方案

1. 不区分能力和开放能力 - 被拒绝，因为会导致调度器误判
2. 只使用 enabledCapabilities - 被拒绝，因为丢失了理论能力信息
