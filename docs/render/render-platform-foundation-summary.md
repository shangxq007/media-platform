# Render Platform Foundation Summary

> **Last Updated:** 2026-06-11
> **Status:** Production-ready core with POC extensions

## 1. Architecture Overview

The Render Platform follows a layered architecture:

```
User Edit (OTIO + metadata.bluepulse)
    │
    ▼
OTIOTimelineCompiler → OTIOTimelineSummary
    │
    ▼
RenderPlanner → RenderPlan (steps, providers, capabilities)
    │
    ▼
RenderOrchestrator → RenderJobFontPreflight → Step Execution → RenderExecutionTrace
    │
    ▼
Providers (FFmpeg, Remotion, GPAC, MLT, Libass, Blender, ...)
```

## 3. Production Provider

**FFmpegRenderProvider** is the only Production-status provider.

Capabilities:
- trim, transcode, mux, demux, extract_audio, thumbnail, output_normalize
- Standard output: MP4/H.264/AAC/fixed fps/fixed resolution/unified metadata
- All other providers' output should pass through FFmpeg for normalization

## 6. RenderJob → RenderPlan → RenderExecutionTrace Flow

```
RenderJob
    │
    ├─ RenderJobFontPreflight
    │   ├─ Check FontAsset status (READY / READY_WITH_SUBSETS)
    │   ├─ Check security.scanStatus == PASSED
    │   ├─ Check security.productionSafe (strict in production mode)
    │   ├─ Check validation.validationStatus == PASSED
    │   ├─ MissingGlyphDetector → missingGlyphs
    │   └─ FontStackResolver (if allowFallback=true)
    │
    ├─ RenderPlanner.plan(job) → RenderPlan
    │   ├─ For each step: select provider by enabledCapabilities
    │   ├─ Respect status/priority/autoDispatch/notFor/mode
    │   └─ Record provider hit reason
    │
    ├─ RenderOrchestrator.execute(plan) → RenderExecutionTrace
    │   ├─ For each step:
    │   │   ├─ Execute provider.render()
    │   │   ├─ Generate RenderStepResult
    │   │   ├─ Generate RenderArtifact (createdByStepId)
    │   │   └─ Record logs/warnings/errors
    │   └─ Final output → RenderArtifactType.FINAL_OUTPUT
    │
    └─ RenderExecutionTrace
        ├─ stepResults[]
        ├─ allArtifacts[]
        ├─ overallSuccess
        └─ fallbackOccurred
```

## 7. OTIO and RenderGraph Boundary

- **OTIO** is the Canonical Edit Timeline / Edit Decision Source of Truth
- **OTIOTimelineCompiler** compiles OTIO + metadata.bluepulse → OTIOTimelineSummary
- **OTIOTimelineSummary** is NOT a RenderGraph
- **RenderPlan** is the execution产物 generated by RenderPlanner from RenderJob + ProviderRegistry
- OTIO does NOT contain provider commands, FFmpeg commands, MLT XML, or LiteFlow chains
- Large data (captions, fonts, templates, effects) are stored externally; OTIO only holds refs

## 8. Font Pipeline Boundary

```
Font Upload
    │
    ▼
QUARANTINED → Security Scan (BasicFontSecurityScanner)
    │
    ▼
SECURITY_PASSED → Validation (NoopFontValidator / FontBakery)
    │
    ▼
READY → Subsetting (NoopFontSubsetter / Pyftsubset)
    │
    ▼
READY_WITH_SUBSETS → FontManifestGenerator → FontManifest
    │
    ▼
RenderJobFontPreflight → FontManifestResolver → ResolvedFont
    │
    ▼
Remotion/FFmpeg/Libass use ResolvedFont (subsetUrl, hash, family)
```

## 9. Remotion Provider Boundary

- Remotion is a **Subtitle & Template Render Provider**
- Does NOT handle: trim, transcode, extract_audio, format repair
- Only responds to: caption_effects, caption_burn_in, template_render
- Uses FontManifestResolver for all font access (never raw sourceUrl)
- Frontend Remotion Player and backend Remotion Renderer share:
  - Same Composition
  - Same inputProps
  - Same FontManifest / subsetUrl
  - Same font assets

## 11. Frozen Items (Not Currently Pursued)

The following items are explicitly frozen and will not be worked on without explicit product decision:

| Item | Reason |
|------|--------|
| BMF real implementation | Spike research only; no proven advantage over FFmpeg/MLT |
| Natron | Overlaps with Blender/Remotion; no node-based VFX demand |
| Real OFX | Current implementation is Java2D simulation; misleading name |
| GStreamer production | System is offline editing focused; no real-time streaming demand |
| Complex LiteFlow rule center | Simple switch + optional LiteFlow for complex workflows only |
| Complete Blender template system | POC only; no 3D template product demand |
| Complete font management backend | Basic upload/scan/subset sufficient for now |
| Multi-template market | Single template support sufficient for now |

## 12. Risk List

| Risk | Impact | Mitigation |
|------|--------|------------|
| Remotion CLI not available in CI | LOCAL smoke test fails | Smoke test is local-only; CI uses MOCK |
| Font tool dependencies missing | Font validation degrades | Noop implementations with clear warnings |
| OTIO library dependency adds complexity | Build complexity | OTIO compiler is skeleton; real OTIO parsing is future |
| Provider capability overlap | Scheduling confusion | ProviderEligibility rules enforce strict boundaries |
| Font subset cache invalidation | Stale font data | Cache key includes fontHash + charsHash + optionsHash |
| Render artifact storage growth | Disk pressure | Artifact retention policy needed in future |
| Missing integration test coverage | Regression risk | LOCAL smoke test + E2E mock test provide baseline |
