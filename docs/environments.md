# Multi-Environment Deployment Guide

## Environments

| Environment | Namespace | Purpose | Auto-Deploy |
|-------------|-----------|---------|-------------|
| **staging** | `media-platform-staging` | Pre-production validation | On merge to `main` |
| **production** | `media-platform` | Live production | Manual approval required |

## Architecture

```
CI Pipeline:
  Build → Push Images → Render Staging Manifests → [Manual Gate] → Render Production Manifests

Deployment:
  staging:  kubectl apply -f build/k8s/staging/
  production: kubectl apply -f build/k8s/production/
```

## Image Tag Strategy

- **Never use `:latest` or `:dev` in staging/production.**
- All images use immutable tags: `git-<shortSHA>` (e.g., `git-abc1234`).
- Tags are generated by CI and injected into manifests at render time.

```bash
# Local verification
REGISTRY=registry.example.com IMAGE_TAG=git-abc1234 ./scripts/render-k8s-manifests.sh staging
REGISTRY=registry.example.com IMAGE_TAG=git-abc1234 ./scripts/render-k8s-manifests.sh production
```

## Environment Differences

| Dimension | Staging | Production |
|-----------|---------|------------|
| namespace | `media-platform-staging` | `media-platform` |
| api replicas | 1 | 2 |
| render-worker replicas | 1 | 2 |
| sandbox-worker replicas | 1 | 1 |
| api resources | 125m CPU / 256Mi mem | 256m CPU / 512Mi mem |
| ingress host | `staging.api.example.com` | `api.example.com` |
| sandbox mode | `external` | `external` |
| dev-auth | `false` | `false` |
| OIDC bootstrap | `false` | `false` |
| log level | `DEBUG` | `INFO` |

## Deployment Steps

### Staging

```bash
# Render manifests
REGISTRY=registry.example.com IMAGE_TAG=git-abc1234 \
  ./scripts/render-k8s-manifests.sh staging

# Review
cat build/k8s/staging/manifests.yaml | grep -E "image:|replicas:|resources:"

# Deploy
kubectl apply -f build/k8s/staging/manifests.yaml

# Verify
kubectl -n media-platform-staging get pods
kubectl -n media-platform-staging get ingress
```

### Production

```bash
# Render manifests (only from main branch)
REGISTRY=registry.example.com IMAGE_TAG=git-abc1234 \
  ./scripts/render-k8s-manifests.sh production

# Review
cat build/k8s/production/manifests.yaml | grep -E "image:|replicas:|resources:"

# Deploy (requires manual approval)
kubectl apply -f build/k8s/production/manifests.yaml

# Verify
kubectl -n media-platform get pods
kubectl -n media-platform get ingress
```

## Secrets

Secrets are **never** committed to Git. Create them manually or via your secrets manager:

```bash
# Create namespace-specific secrets
kubectl create secret generic platform-secrets \
  --from-literal=SPRING_DATASOURCE_PASSWORD='<db-password>' \
  --from-literal=APP_JWT_SECRET='<jwt-secret-min-256-bits>' \
  --from-literal=APP_REMOTE_WORKER_API_KEY='<worker-key>' \
  --from-literal=STRIPE_WEBHOOK_SECRET='<stripe-secret>' \
  --from-literal=HYPER_SWITCH_WEBHOOK_SECRET='<hs-secret>' \
  -n media-platform

# Same for staging with staging-specific values
kubectl create secret generic platform-secrets \
  --from-literal=SPRING_DATASOURCE_PASSWORD='<staging-db-password>' \
  --from-literal=APP_JWT_SECRET='<staging-jwt-secret>' \
  --from-literal=APP_REMOTE_WORKER_API_KEY='<staging-worker-key>' \
  --from-literal=STRIPE_WEBHOOK_SECRET='<staging-stripe-secret>' \
  --from-literal=HYPER_SWITCH_WEBHOOK_SECRET='<staging-hs-secret>' \
  -n media-platform-staging
```

## TLS

TLS certificates are referenced via `secretName` in the Ingress. Create them:

```bash
# Production
kubectl create secret tls api-tls \
  --cert=certs/api.example.com.crt \
  --key=certs/api.example.com.key \
  -n media-platform

# Staging
kubectl create secret tls staging-api-tls \
  --cert=certs/staging.api.example.com.crt \
  --key=certs/staging.api.example.com.key \
  -n media-platform-staging
```

## Rollback

```bash
# Check rollout history
kubectl rollout history deployment/api -n media-platform

# Rollback to previous revision
kubectl rollout undo deployment/api -n media-platform

# Or deploy a specific previous image tag
REGISTRY=registry.example.com IMAGE_TAG=git-prevsha123 \
  ./scripts/render-k8s-manifests.sh production
kubectl apply -f build/k8s/production/manifests.yaml
```

## Production Checklist

Before deploying to production, verify:

- [ ] Image tag is immutable (git SHA), not `latest` or `dev`
- [ ] `app.security.dev-auth-endpoint=false`
- [ ] `app.security.oidc-dev-bootstrap=false`
- [ ] `sandbox.execution-mode=external` (not `in-process`)
- [ ] All containers have `securityContext.runAsNonRoot: true`
- [ ] All containers have `securityContext.readOnlyRootFilesystem: true`
- [ ] All containers have `capabilities.drop: [ALL]`
- [ ] Resource requests and limits are set
- [ ] Replicas >= 2 for api and render-worker
- [ ] Ingress uses production domain
- [ ] TLS secret exists
- [ ] Database secret exists
- [ ] NetworkPolicy for sandbox-worker is applied
- [ ] NetworkPolicy for platform-api egress is applied
- [ ] NetworkPolicy for render-worker egress is applied
- [ ] No `0.0.0.0/0` in any egress NetworkPolicy
- [ ] No `169.254.169.254` in any egress NetworkPolicy
- [ ] Egress proxy configured for external dependencies (OIDC, webhook, S3, LiteLLM)
- [ ] Egress proxy deployed (deployment-egress-proxy, service-egress-proxy)
- [ ] Egress proxy NetworkPolicy blocks sandbox-worker ingress
- [ ] Egress proxy Squid config blocks metadata IP (169.254.169.254)
- [ ] Egress proxy allowed-domains.txt configured with production domains (no example.com placeholders)
- [ ] Egress proxy Squid uses `http_access deny all` (no open proxy)
- [ ] HTTP_PROXY/HTTPS_PROXY/NO_PROXY env vars set for api and render-worker
- [ ] sandbox-worker has NO proxy env vars
- [ ] Egress proxy smoke test enabled in staging before production
- [ ] Smoke URL configured and in Squid allowed-domains
- [ ] JVM proxy properties assessed (enable if Java clients don't honor env proxy)
- [ ] Staging smoke rollout completed per [Egress Smoke Rollout Runbook](egress-smoke-rollout.md)
- [ ] S3/OIDC/JWKS/webhook/provider verified through proxy in staging
- [ ] allowed-domains.txt has real domains (no example.com placeholders)
- [ ] Production promotion criteria met per runbook
