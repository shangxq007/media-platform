# 问题数据处理说明

> **最后更新:** 2026-05-14

---

## 概述

问题数据检测与处理系统自动识别由系统 bug 导致的数据问题和行为异常。提供统一的检测 → 隔离 → 自动修复 → 人工复核 → 解决流程。

---

## 检测范围

| 数据源 | 检测规则数 | 覆盖范围 |
|--------|-----------|----------|
| 渲染任务 (RenderJob) | 5 | 缺失输出、卡住任务、重复、SLA 违规、成本异常 |
| 提示词执行 (PromptExecution) | 3 | 数据泄露、输出不匹配、风险升级 |
| 提供者/工作者 | 2 | 错误率飙升、心跳超时 |
| KPI/SLA | 2 | SLA 违规、成本异常 |

---

## 检测规则

### 渲染任务规则

| 规则 ID | 类型 | 严重度 | 自动修复 | 说明 |
|---------|------|--------|----------|------|
| RJB-001 | 缺失字段 | 高 | 否 | 渲染任务完成但无输出工件 |
| RJB-002 | 无效状态转换 | 中 | 是 | 任务在非终端状态卡住超过 30 分钟 |
| RJB-003 | 重复条目 | 低 | 是 | 相同项目+预设+时间线的重复任务 |
| SLA-001 | SLA 违规 | 严重 | 否 | 渲染任务超过 SLA 时间限制 |
| CST-001 | 成本异常 | 高 | 否 | 实际成本超过预估成本 2 倍 |

### 提示词执行规则

| 规则 ID | 类型 | 严重度 | 自动修复 | 说明 |
|---------|------|--------|----------|------|
| PMT-001 | 数据泄露 | 严重 | 否 | 执行记录中发现敏感变量 |
| PMT-002 | 输出不匹配 | 高 | 否 | 输出与预期格式不符 |
| PMT-003 | 风险升级 | 高 | 否 | 执行后风险等级上升 |

### 提供者/工作者规则

| 规则 ID | 类型 | 严重度 | 自动修复 | 说明 |
|---------|------|--------|----------|------|
| PRV-001 | 错误率飙升 | 高 | 否 | 错误率超过 20% |
| WRK-001 | 心跳超时 | 中 | 是 | 工作者心跳超过 5 分钟未更新 |

## 处理流程

```
检测 → 标记 → 自动修复/隔离 → 人工复核 → 解决
  │       │          │              │        │
  │       │          │              │        │
  ▼       ▼          ▼              ▼        ▼
DETECTED → AUTO_FIXING → AUTO_FIXED → HUMAN_REVIEW → RESOLVED
                    │                    │
                    ▼                    ▼
              QUARANTINED          FALSE_POSITIVE
```

### 自动修复范围

| 问题类型 | 自动修复 | 修复动作 |
|----------|----------|----------|
| 缺失字段 | ✅ 是 | 填充默认值 |
| 格式错误 | ✅ 是 | 转换为标准格式 |
| 重复条目 | ✅ 是 | 标记为重复，保留原始 |
| 卡住任务 | ✅ 是 | 重置为 QUEUED 重试 |
| 工作者心跳超时 | ✅ 是 | 标记为离线，重新分配任务 |
| SLA 违规 | ❌ 否 | 需要业务影响评估 |
| 输出不匹配 | ❌ 否 | 需要重新渲染 |
| 成本异常 | ❌ 否 | 需要定价模型审查 |
| 密钥泄露 | ❌ 否 | 需要安全事件响应 |

### 隔离策略

| 严重度 | 动作 |
|--------|------|
| 严重 | 立即隔离 + Sentry 告警 |
| 高 | 隔离 + 通知 |
| 中 | 标记待审查，继续处理 |
| 低 | 记录日志，尝试自动修复 |

## 数据库表

| 表名 | 用途 |
|------|------|
| `problematic_data_record` | 主表，记录所有检测到的异常 |
| `quarantined_render_jobs` | 隔离的渲染任务数据 |
| `quarantined_prompt_executions` | 隔离的提示词执行数据 |
| `quarantined_provider_workers` | 隔离的提供者/工作者数据 |
| `problematic_data_rule_config` | 检测规则配置 |

## 审计集成

所有操作写入审计跟踪：

| 事件 | 类别 | 详情 |
|------|------|------|
| `PROBLEMATIC_DATA_DETECTED` | AUDIT | 数据类型、严重度、规则 |
| `PROBLEMATIC_DATA_AUTO_FIXED` | AUDIT | 数据类型、修复动作、状态 |
| `PROBLEMATIC_DATA_QUARANTINED` | AUDIT | 数据类型、隔离表、严重度 |

---

## 重跑/重试流程

1. 审查隔离记录
2. 确定根本原因
3. 如可重跑：重置任务状态为 QUEUED，清除隔离记录
4. 如不可重跑：标记为 RESOLVED，记录解决说明，通知用户
