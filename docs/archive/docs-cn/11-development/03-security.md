# 安全策略

> **最后更新：** 2026-05-18

## 认证

**当前状态：** 通过 `ApiKeyAuthFilter` 进行 API Key 认证

| 请求头 | 用途 |
|--------|------|
| `X-API-Key` | 服务间认证的 API 密钥 |
| `X-Tenant-ID` | 租户标识 |
| `X-User-ID` | 用户标识 |

**🔴 生产阻塞项：** 未为生产配置 Spring Security 过滤器链。JWT 认证未实现。

## API 密钥安全

- API 密钥以 SHA-256 哈希存储（永不存储明文）
- 存储密钥指纹用于识别
- 密钥可撤销
- 跟踪最后使用时间

## 租户隔离

**🔴 生产阻塞项：** `TenantContext` 存在但未在数据层强制执行。所有查询必须手动限定到当前租户。

## CORS

| 环境 | 策略 |
|------|------|
| 开发环境 | `*`（所有来源）|
| 生产环境 | 可配置白名单 |

## 速率限制

| 套餐 | 请求/分钟 | 突发 |
|------|-----------|------|
| 免费版 | 30 | 10 |
| 专业版 | 120 | 30 |
| 团队版 | 600 | 100 |
| 企业版 | 无限制 | 无限制 |

## 进程执行安全

- ✅ 业务代码中禁止使用 `Runtime.exec()` 或 `ProcessBuilder`
- ✅ JavaCV 直接使用 JNI 绑定
- ✅ 扩展模块使用基于白名单的 CLI 执行
- ✅ 路径遍历保护
- ✅ 空字节注入防护
- ⚠️ `extension-module` 中仍存在 Apache Commons Exec

## 数据保护

- 日志中敏感数据脱敏
- Sentry/OpenReplay 中敏感数据脱敏
- NLQ 查询结果中 PII 字段脱敏
- API 密钥永不记录
