# Release Candidate: P4 Import/Export Pipeline

**Date:** 2026-06-06
**Tag:** rc/p4-import-export-2026-06-06
**Status:** RC only - not production-ready

## Scope

This RC covers the P4 Import/Export pipeline for the audio/video editing platform:

1. **Project Export** - JSON and ZIP export with metadata_only and linked_assets modes
2. **Project Import Preview** - JSON and ZIP preview without side effects
3. **Project Import Execute Shell** - ZIP shell import creating project shell with metadata persistence
4. **Import Metadata Read API** - Summary and detail endpoints for persisted import metadata
5. **Frontend Metadata Preview** - UI panel showing imported metadata status
6. **Security Hardening** - Tenant isolation, URL scrubbing, audit safety, zip security

## Completed Capabilities

### Export
- JSON export (metadata_only, linked_assets)
- ZIP export archive endpoint
- Signed URL generation for linked_assets (3600s default TTL, 86400s max)
- Checksum validation (sha256sums.txt)
- Zip slip/bomb protection
- Filename sanitization

### Import Preview
- JSON import preview
- ZIP import preview with ProjectExportZipReader
- Entry allowlist validation
- Checksum verification
- No DB writes, no URL downloads

### Import Execute Shell
- ZIP shell import creating project shell
- All assets marked needs_upload
- Metadata persistence (timeline, render, spatial, effects, asset mapping)
- Transactional rollback on failure
- Audit with redaction guard

### Metadata Read API
- Summary endpoints (boolean flags only)
- Detail endpoints (scrubbed JSON content)
- Tenant-scoped queries
- Defensive scrub-on-read

### Frontend
- ImportedMetadataPanel integrated into ExportPanel
- Defense-in-depth frontend sanitization
- No editor timeline mutation
- No media download/upload

## Security Guarantees

- **Tenant isolation:** Path tenantId is sole trusted source; source tenantId from zip is ignored
- **URL safety:** No signed URLs in audit; storageUri/storageRef never exposed
- **Zip security:** Zip slip/bomb protection, checksum validation, entry allowlist
- **Scrub-on-write + scrub-on-read:** Metadata scrubbed before DB save and on read
- **Audit safety:** Payloads contain only safe fields (no URLs, no JSON content, no zip bytes)
