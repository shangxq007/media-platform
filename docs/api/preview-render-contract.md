# Preview Render Contract — Frontend Integration Guide

**Date:** 2026-07-06
**Status:** STABLE
**Authority:** RENDER-PREVIEW-FLOW.0

---

## Overview

The preview render flow allows frontend clients to create render jobs, track status, and retrieve artifact metadata.

## Endpoints

### 1. Create Render Job

```
POST /api/v1/render/jobs
```

**Request:**
```json
{
  "tenantId": "tenant-1",
  "projectId": "project-123",
  "prompt": "optional timeline JSON or AI prompt",
  "profile": "default_1080p",
  "timelineSnapshotId": "optional-snapshot-id"
}
```

**Response (201):**
```json
{
  "id": "rj-abc123",
  "projectId": "project-123",
  "timelineSnapshotId": "snap-xyz",
  "profile": "default_1080p",
  "status": "CREATED"
}
```

### 2. Get Render Job Status

```
GET /api/v1/render/jobs/{jobId}
```

**Response (200):**
```json
{
  "jobId": "rj-abc123",
  "tenantId": "tenant-1",
  "projectId": "project-123",
  "status": "SUCCEEDED",
  "outputProductId": "prod-output-456",
  "createdAt": "2026-07-06T10:00:00Z",
  "completedAt": "2026-07-06T10:05:00Z"
}
```

**Status values:**
- `CREATED` — Job created, not yet queued
- `QUEUED` — Waiting for execution
- `RUNNING` — Currently executing
- `SUCCEEDED` — Completed successfully
- `FAILED` — Execution failed
- `CANCELLED` — Cancelled by user

### 3. Get Artifact Metadata

```
GET /api/v1/artifacts/{artifactId}
```

**Response (200):**
```json
{
  "artifactId": "art-789",
  "renderJobId": "rj-abc123",
  "projectId": "project-123",
  "mimeType": "video/mp4",
  "outputFormat": "mp4",
  "width": 1920,
  "height": 1080,
  "fps": 30,
  "durationSeconds": 120.5,
  "hasSubtitles": true,
  "createdAt": "2026-07-06T10:05:00Z"
}
```

### 4. List Render Jobs

```
GET /api/v1/render/jobs?projectId={projectId}
```

**Response (200):** Array of render job objects.

## Error Response Format

```json
{
  "error": "RENDER_JOB_NOT_FOUND",
  "message": "Render job not found: rj-xxx",
  "timestamp": "2026-07-06T10:00:00Z"
}
```

## Frontend Integration Sequence

```
1. POST /api/v1/render/jobs        → Get jobId
2. GET  /api/v1/render/jobs/{id}   → Poll status
3. GET  /api/v1/artifacts/{id}     → Get artifact metadata
```

---

## Current Limitations

- Actual FFmpeg render execution requires backend infrastructure
- Preview mode may use placeholder execution
- No real-time WebSocket status updates (polling only)
