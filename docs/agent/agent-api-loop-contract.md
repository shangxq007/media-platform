## Overview

This document defines the agent-driven API execution loop for media-platform.

## Canonical Loop

```
Agent
  ├─ 1. Health Check
  ├─ 2. Auth (if required)
  ├─ 3. Create Render Job (opt-in)
  ├─ 4. Poll Job Status
  ├─ 5. Fetch Artifact Metadata
  └─ 6. Return Structured Result
```

## Job Status Lifecycle

| Status | Terminal | Description |
|--------|----------|-------------|
| CREATED | No | Job created |
| QUEUED | No | Waiting for execution |
| RUNNING | No | Currently executing |
| SUCCEEDED | Yes | Completed successfully |
| FAILED | Yes | Execution failed |
| CANCELLED | Yes | Cancelled by user |

## API Endpoints

| Step | Method | Endpoint |
|------|--------|----------|
| Health | GET | `/actuator/health` |
| Create Job | POST | `/api/v1/render/jobs` |
| Get Status | GET | `/api/v1/render/jobs/{jobId}` |
| List Jobs | GET | `/api/v1/render/jobs` |
| Get Artifact | GET | `/api/v1/artifacts/{artifactId}` |

## Safety Rules

- Read-only mode is default
- Write mode requires `AGENT_API_WRITE=1`
- No destructive operations (delete, purge)
- No admin/internal APIs
- No secrets in output
- Token via env var only, never committed
